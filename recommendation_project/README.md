# 基于特征的深度学习餐厅推荐系统
本项目是一个基于 PyTorch 实现的深度学习推荐系统，旨在根据用户的历史行为和属性，以及商家的详细特征，来预测用户对餐厅的评分。

该项目的核心特点是采用了**纯粹基于特征（Feature-Based）**的架构，完全摒弃了对用户或商家ID的嵌入（Embedding），使其天然具备了处理新用户和新商家（即冷启动问题）的强大能力，具有极高的鲁棒性和现实应用价值。

核心思路与特性 (Core Philosophy)
纯粹基于特征: 模型的预测能力完全来自于对用户和商家自身属性的学习，而不是记忆特定的ID。这使得模型能够从特征中泛化，而不是过度拟合到已知的交互上。

鲁棒的冷启动处理: 由于不依赖ID嵌入，任何新用户或新商家只要具备特征，就能被模型立即处理并生成有意义的推荐，无需重新训练。

专用编码器 (Specialized Encoders): 为用户和商家分别设计了独立的编码器 (UserEncoder 和 BusinessEncoder)，允许对不同类型的特征进行专门的、非对称的处理和优化。

灵活的交互层: 采用“拼接+MLP”的方式来融合用户和商家的最终向量。相比于传统的点积，这种方式能够学习两者之间更复杂、更深层次的非线性关系。

配置驱动: 整个项目由 configs/config.yaml 文件驱动，无论是模型结构、超参数还是文件路径，都可以通过修改配置文件轻松调整，无需改动源代码。

项目架构 (Architecture)
本系统采用了一个先进的双塔模型（Two-Tower Model）架构，但其核心在于塔的构建和塔顶的交互方式。
```
+--------------------------+                                      +-----------------------------+
|      用户原始特征         |                                      |      商家原始特征            |
| (review_count, fans,     |                                      | (stars, review_count,      |
|  account_age, etc.)      |                                      |  categories, etc.)         |
+--------------------------+                                      +-----------------------------+
             |                                                                    |
             ▼                                                                    ▼
+--------------------------+                                      +-----------------------------+
|       UserEncoder        |                                      |      BusinessEncoder        |
| (一个专用的MLP网络)        |                                      | (包含EmbeddingBag处理类别)    |
+--------------------------+                                      +-----------------------------+
             |                                                                    |
             ▼                                                                    ▼
+--------------------------+                                      +-----------------------------+
|    用户嵌入向量 (User)     |                                      |    商家嵌入向量 (Item)       |
|       (e.g., 32-dim)     |                                      |       (e.g., 32-dim)        |
+--------------------------+                                      +-----------------------------+
             |                                                                    |
             +------------------------------+-------------------------------------+
                                            |
                                            ▼
                               +-----------------------------+
                               |     拼接 (Concatenate)      |
                               |  (e.g., 64-dim vector)      |
                               +-----------------------------+
                                            |
                                            ▼
                               +-----------------------------+
                               |   交互层 (Interaction MLP)  |
                               | (学习复杂的非线性关系)        |
                               +-----------------------------+
                                            |
                                            ▼
                               +-----------------------------+
                               |      最终预测评分 (1-5)      |
                               +-----------------------------+
```
用户塔 (User Tower): UserEncoder 接收一系列标准化的用户数值特征，通过一个多层感知机（MLP）网络，将其编码成一个低维稠密的“用户嵌入向量”。

商家塔 (Item Tower): BusinessEncoder 分别处理商家的数值特征和类别（categories）特征。它使用 nn.EmbeddingBag 高效地将多个类别融合成一个向量，然后与数值特征拼接，再一同送入MLP，最终生成“商家嵌入向量”。

交互层 (Interaction Layer): 将用户和商家两个向量拼接起来，形成一个更长的向量，然后将其输入到一个新的、独立的MLP网络中。这个网络负责学习两者之间复杂的交互模式，并最终输出一个单一的预测评分。

如何使用 (Workflow)
请按照以下步骤来运行完整的项目流程。

1. 环境设置
首先，克隆本项目，并使用附带的Python虚拟环境。

```Bash
# 建议使用 Python 3.8+
source env/bin/activate  # On Windows, use `venv\Scripts\activate`
```


2. 数据准备
将Yelp数据集的原始JSON文件（user.json, business.json, review.json等）放置在 data/unprocessed/ 目录下。

3. 完整的处理与训练流程
在项目根目录下，按顺序执行以下命令：


第1步 - 训练模型 执行主训练脚本。它会自动读取配置文件，加载训练数据，构建模型，并开始训练。训练过程中最好的模型将被保存在 saved_models/best_model.pt。

```Bash
python src/train.py
```

第2步 - 评估模型 训练完成后，运行评估脚本。它会加载 best_model.pt 并在测试集上计算性能指标（如RMSE和MAE）。

```Bash
python src/evaluate.py
```
